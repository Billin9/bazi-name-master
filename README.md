# bazi-name-master

`bazi-name-master` 是一个面向支持 Skills 的 Agent 场景的八字起名项目，适用于 Claude Code、Codex、Gemini CLI、OpenClaw 等能够加载和调用 skill 的智能体环境，用于把“出生信息收集、四柱计算核验、命局分析、候选名评估”串成一条可执行、可复核的工作流。

这个项目的目标不是泛泛给名字灵感，而是让命名过程先立盘、再辨病药、最后评估名字是否真正对盘。

## 项目内容

- `SKILL.md`：技能主说明，定义触发条件、对话路由、子任务职责和输出结构。
- `scripts/calculate_bazi.py`：统一的八字计算脚本，使用 `lunar-python` 进行排盘与四柱一致性核验。
- `references/`：命名原则、资料优先级、平台风格研究等参考资料。
- `evals/evals.json`：用于验证技能行为是否符合预期的评测样例。

## 适用场景

- 用户只提供出生年月日时、历法和出生地，需要先计算八字。
- 用户已经给出四柱八字，希望直接分析命局与命名方向。
- 用户已经有候选名字，希望结合已核验的八字做拆字解读、打分和排序。

## 三种命名模式

### 1. 先算八字，再进入命名分析

适用于用户只提供原始出生信息，还没有现成四柱结果的情况。  
这个模式会先收集姓氏、性别、出生年月日时、历法、出生地等基础信息，再调用脚本排盘，确认四柱后进入命局分析和命名方向判断。

### 2. 已有八字，直接看命名方向

适用于用户已经给出完整四柱，想直接分析日元旺衰、病药、喜用忌用和命名补益策略的情况。  
如果用户同时也给了原始出生信息，skill 会优先做一次一致性核验，避免后续所有判断建立在错误盘面上。

### 3. 已有候选名字，做适配评估与排序

适用于用户已经有一个或多个候选名字，希望判断这些名字与八字是否真正相配的情况。  
这个模式要求八字已经完成核验，然后再对候选名逐个做拆字解读、字义意象分析、音律字形评估、命理适配说明和固定维度评分排序。

## 工作流概览

1. 收集必要信息，确认姓氏、性别、出生时间、历法、出生地等基础输入。
2. 使用脚本统一计算四柱，必要时与用户自带四柱做一致性核验。
3. 基于核验后的八字分析日元旺衰、病药、喜用忌用和命名补益方向。
4. 对候选名字进行字义、音律、字形与命理适配度评估。

## 八字脚本使用方式

默认情况下，不需要手动执行脚本。  
在支持 Skills 的 Agent 环境里，直接调用这个 skill 即可，skill 会根据用户输入自动判断流程，并在需要时调用 `scripts/calculate_bazi.py` 完成排盘或一致性核验。

手动执行脚本只适用于以下场景：

- 你在独立调试脚本输出
- 你在校验某个具体出生信息的排盘结果
- 你在开发或维护这个项目，需要验证脚本行为

本项目脚本默认通过 `uv` 运行，下面的命令主要用于调试和维护。

### 公历示例

```bash
uv run /Users/Bai/.agents/skills/bazi-name-master/scripts/calculate_bazi.py \
  --calendar solar \
  --date 2024-04-22 \
  --time 13:30 \
  --birthplace "江苏南京"
```

### 农历示例

```bash
uv run /Users/Bai/.agents/skills/bazi-name-master/scripts/calculate_bazi.py \
  --calendar lunar \
  --date 2024-03-14 \
  --time 13:30 \
  --birthplace "江苏南京"
```

### 一致性核验示例

```bash
uv run /Users/Bai/.agents/skills/bazi-name-master/scripts/calculate_bazi.py \
  --calendar solar \
  --date 2024-04-22 \
  --time 13:30 \
  --birthplace "江苏南京" \
  --expected-pillars "甲辰,戊辰,丙辰,乙未"
```

## 设计原则

- 先核盘，再分析，不在信息不完整时抢跑下结论。
- 命局判断优先于审美修辞。
- 名字推荐必须先过补益与避讳筛选，再比较出处、音律与现代审美。
- 候选名字评估必须基于已核验的八字结果，不允许跳过前置计算。
